# 2026-06-29 개발 메모

## 작업 주제

30일 데이터에서 보였던 방향성 예측/고변동 예측 쏠림 현상이 데이터 부족 때문인지 확인하기 위해, 365일 데이터를 수집하고 전체 파이프라인을 다시 실행했다.

## 1. 리스크 타깃 생성

`create_risk_targets.py`를 추가했다.

목적:

```text
단기 방향성 예측이 아니라,
다음 1시간 동안의 큰 움직임, 급락, 급등, 고변동 여부를 분류하기 위한 target 생성
```

입력 파일:

```text
data/processed/BTCUSDT_15m_features.csv
```

출력 파일:

```text
data/processed/BTCUSDT_15m_risk_targets.csv
```

생성한 주요 타깃:

```text
target_return_next_4
target_abs_return_next_4
target_volatility_next_4
target_big_move_next_4
target_drop_next_4
target_pump_next_4
target_volatility_high_next_4
```

타깃 의미:

```text
next_4 = 다음 4개 15분봉 = 다음 1시간
```

30일 데이터 기준 class ratio:

```text
target_big_move_next_4: 3.74%
target_drop_next_4: 1.66%
target_pump_next_4: 2.08%
target_volatility_high_next_4: 29.99%
```

해석:

- 급락/급등/큰 움직임은 희귀 이벤트라 바로 모델링하기 어렵다.
- `target_volatility_high_next_4`는 약 30% 양성 비율이라 첫 리스크 모델로 적합하다.

## 2. 고변동 예측 모델 생성

`train_volatility_classifier.py`를 추가했다.

목적:

```text
다음 1시간 동안 고변동장이 올 가능성을 예측하는 분류 모델 생성
```

중요한 설계:

- 직접 매수/매도 신호가 아니라 리스크 관리용 모델로 정의했다.
- 포지션 축소, 신규 진입 금지, 전략 선택, Tilt Guard에 활용하는 것을 목표로 했다.
- 시간순 70/15/15 train/validation/test split을 사용했다.
- shuffle은 사용하지 않았다.
- XGBoostClassifier를 사용했다.
- train 데이터 기준으로 `scale_pos_weight`를 계산했다.

30일 데이터 기준 결과:

```text
전체 row: 1,927
feature 수: 26
train high vol ratio: 28.93%
validation high vol ratio: 29.41%
test high vol ratio: 35.52%
scale_pos_weight: 2.456410

accuracy: 0.406897
precision: 0.364706
recall: 0.902913
f1: 0.519553
roc_auc: 0.537719
average_precision: 0.461486
log_loss: 1.034779
confusion matrix: [[25, 162], [10, 93]]
```

해석:

- 고변동을 많이 잡긴 하지만, 오탐도 많았다.
- 30일 기준 모델은 고변동을 과하게 경고하는 성격이었다.
- 확률 분포가 0.8~0.95에 많이 몰렸다.

## 3. 365일 데이터 재실험

30일 결과가 짧은 데이터 때문인지 확인하기 위해 365일 데이터를 다시 수집했다.

365일 데이터 수집 결과:

```text
raw 1m rows: 525,075
기간: 2025-06-29 13:56 UTC ~ 2026-06-29 13:54 UTC
15m resampled rows: 35,041
15m feature rows: 34,087
risk target rows: 34,083
```

주의점:

```text
raw 1분봉에서 2분 간격 gap이 524개 있었음
resampled 5m 이상에서는 interval issue가 0
```

현재 15분봉 기반 분석에는 큰 문제는 아닌 것으로 판단했다.

## 4. 30일 vs 365일 핵심 비교

### 방향성 모델

```text
pred_label=1 비율: 88.62% → 73.45%
pred_proba_up 평균: 약 0.684 → 0.561657
roc_auc: 0.606849 → 0.514879
```

해석:

- 상승 예측 쏠림은 완화됐다.
- 하지만 AUC가 거의 랜덤 수준으로 떨어졌다.
- 30일에서 좋아 보였던 방향성 성능은 특정 기간 장세 영향이 컸을 가능성이 높다.

### 방향성 백테스트

```text
30일 best total return: -3.97%
365일 best total return: -11.09%
365일 test 구간 buy & hold: -27.11%
```

해석:

- 365일 test 구간에서는 Buy & Hold보다 덜 잃었다.
- 하지만 방향성 모델 AUC가 약해 단순 방향성 예측 전략으로 보기에는 한계가 있다.
- 수수료, 거래 횟수, 노이즈 문제는 여전히 중요하다.

### 고변동 모델

```text
roc_auc: 0.537719 → 0.778481
average_precision: 0.461486 → 0.631077
pred_label=1 비율: 87.93% → 60.38%
```

해석:

- 고변동 예측은 365일 데이터에서 크게 개선됐다.
- 고변동 예측 쏠림도 완화됐다.
- 방향성 모델보다 고변동/리스크 모델이 훨씬 안정적이고 프로젝트 방향에 적합하다.

## 6. 오늘의 결론

365일 재실험 결과, 단기 방향성 예측은 일반화가 약했고 고변동 예측은 데이터가 늘어나며 오히려 성능이 개선됐다.

따라서 프로젝트는 `가격 예측 AI`가 아니라 `시장 위험 국면 탐지 및 리스크 필터링 시스템`으로 발전시키는 것이 더 타당하다.
