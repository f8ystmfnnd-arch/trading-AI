### 2. 데이터 파이프라인 정리

Bybit BTCUSDT 데이터를 수집하고, 여러 타임프레임으로 리샘플링하는 흐름을 만들었다.

주요 데이터 흐름:

```text
1m raw data
→ 5m / 15m / 1h / 4h / 1d resampled data
→ multi-timeframe feature dataset
→ model training
→ backtest
```

주요 파일:

```text
collect_bybit_1m.py
resample_ohlcv.py
check_data_quality.py
create_features_multi_timeframe.py
train_xgboost_multi_timeframe.py
backtest_xgboost_signal.py
```

### 3. 멀티타임프레임 피처 생성

15분봉을 메인 기준으로 사용하고, 1시간봉과 4시간봉 정보를 결합했다.

주요 피처:

```text
return_1, return_3, return_5, return_10
ma_20, ma_60, ma_120
ma_distance_20, ma_distance_60
volatility_20, volatility_60
rsi_14
volume_change_1, volume_ma_20
high_low_range, close_open_return
h1_return_1, h1_ma_20, h1_ma_60, h1_ma_distance_20, h1_trend_direction
h4_return_1, h4_ma_20, h4_ma_60, h4_ma_distance_20, h4_trend_direction
```

중요한 점:

- 1시간봉과 4시간봉 데이터는 해당 봉이 완전히 종료된 이후에만 사용할 수 있도록 timestamp를 close time 기준으로 조정했다.
- `merge_asof(direction="backward")`를 사용해 미래 정보가 섞이지 않도록 했다.
- target 컬럼은 feature에서 제외했다.

### 4. 방향성 XGBoost 모델 학습

`target_up_next`를 기준으로 다음 15분봉 방향성을 예측하는 분류 모델을 학습했다.

30일 데이터 기준 주요 결과:

```text
accuracy: 0.517241
precision: 0.509728
recall: 0.903448
f1: 0.651741
roc_auc: 0.606849
log_loss: 0.771222
```

관찰:

```text
pred_label=1 비율: 약 88.6%
pred_proba_up 평균: 약 0.684
```

해석:

- 모델이 상승을 너무 많이 예측했다.
- AUC는 0.60 수준으로 약간의 신호가 있어 보였지만, 예측이 상승 쪽으로 많이 치우쳐 있었다.
- 30일 데이터만으로는 모델의 일반화 성능을 판단하기 어렵다.

### 5. 방향성 모델 백테스트

`backtest_xgboost_signal.py`를 만들어 방향성 모델의 threshold 기반 long/cash 전략을 테스트했다.

30일 데이터 기준 주요 결과:

```text
best threshold: 0.65
total_return: -3.97%
buy_and_hold_return: -2.30%
max_drawdown: -5.57%
number_of_trades: 113
win_rate: 54.39%
profit_factor: 1.44
```

해석:

- 단순 방향성 모델 기반 전략은 Buy & Hold보다 좋지 않았다.
- 거래 횟수가 너무 많아 수수료 부담이 컸다.
- 단기 방향성 예측만으로 매매 전략을 구성하는 것은 위험해 보인다.

## 오늘의 결론

단기 가격/방향 예측은 차트 데이터만으로는 불안정하고, 외부 변수와 노이즈의 영향을 크게 받는다.

따라서 프로젝트 방향을 단순한 가격 예측 AI가 아니라, 시장 국면과 리스크를 판단하는 방향으로 전환하는 것이 더 타당하다고 판단했다.
