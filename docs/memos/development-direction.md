# 개발 진행 방향성 메모

## 1. 프로젝트 이름

현재 프로젝트의 방향은 다음 이름으로 정리할 수 있다.

**BTC Market Regime & Risk Guard AI**

이 프로젝트는 비트코인의 단기 가격을 정확히 맞히는 것을 목표로 하지 않는다. 대신 시장 국면, 변동성 확대 가능성, 급락 위험, 뉴스 기반 이벤트 위험도를 분석해 매매 전략의 진입 필터와 리스크 관리에 활용하는 것을 목표로 한다.

## 2. 핵심 철학

단기 가격 예측은 외부 변수의 영향을 너무 크게 받는다.

예를 들어 다음과 같은 이벤트는 차트만으로 예측하기 어렵다.

- 금리 발언
- CPI/FOMC 같은 거시 이벤트
- ETF 자금 유입/유출
- 거래소 해킹
- SEC 규제 뉴스
- 지정학 리스크
- 대형 청산
- 스테이블코인 디페깅

따라서 이 프로젝트는 다음 방향을 따른다.

- 가격 자체를 맞히는 AI가 아니다.
- 시장 위험도를 판단하는 AI다.
- 방향성 예측은 보조 신호로만 사용한다.
- 고변동/급락 위험 예측을 리스크 필터로 사용한다.
- 뉴스 기반 심각성 분석을 통해 차트에 반영되기 전의 외부 리스크를 감지한다.
- 최종 목표는 수익률 극대화보다 MDD 감소, 위험 회피, 포지션 조절, 거래 중지 판단이다.

## 3. 전체 시스템 구조

최종 구조는 다음과 같이 설계한다.

```text
[데이터 수집]
- Bybit BTCUSDT 1m 캔들
- 5m / 15m / 1h / 4h / 1d 리샘플링
- 실시간 차트 데이터
- 뉴스 데이터
- 거시 이벤트 데이터
- 추후 funding rate, open interest, liquidation data 추가 가능

        ↓

[피처 생성]
- 멀티타임프레임 가격/수익률 피처
- 이동평균/이격도 피처
- 변동성 피처
- 거래량 피처
- 뉴스 심각도 피처
- 이벤트 위험도 피처

        ↓

[모델]
- 방향성 분류 모델
- 고변동 예측 모델
- 급락 위험 예측 모델
- 시장 국면 분류 모델
- 뉴스 심각성 분석 모델

        ↓

[리스크 엔진]
- chart_risk_score
- news_risk_score
- volatility_risk_score
- drop_risk_score
- regime_score

        ↓

[최종 판단]
- 정상 거래 가능
- 주의
- 포지션 축소
- 신규 진입 금지
- 강제 휴식
- 전략 전환
```

## 4. 매매 스타일 분리

장기투자는 제외하고 다음 두 가지 방식으로 나눈다.

### 4.1 Day Trading Mode

데이트레이딩은 짧은 시간 단위의 진입 위험을 판단한다.

사용 타임프레임:

- 1m
- 5m
- 15m
- 1h 보조

주요 예측 대상:

- 다음 15분 방향성
- 다음 1시간 고변동 가능성
- 다음 1시간 급락 위험
- 당일 변동성 상태

활용 방식:

- 지금 신규 진입해도 되는가?
- 고변동이라 쉬어야 하는가?
- 포지션 크기를 줄여야 하는가?
- 짧은 손절/익절 전략이 적합한가?

### 4.2 Swing Trading Mode

스윙 매매는 며칠 단위의 시장 국면과 포지션 유지 여부를 판단한다.

사용 타임프레임:

- 15m
- 1h
- 4h
- 1d

주요 예측 대상:

- 다음 4시간 변동성
- 다음 1일 변동성
- 다음 1일 급락 위험
- 시장 국면: 상승 / 하락 / 횡보 / 고변동

활용 방식:

- 며칠 들고 갈 만한 시장인가?
- 신규 스윙 진입이 가능한 환경인가?
- 기존 포지션을 유지할지 축소할지 판단
- 급락 위험이 커졌을 때 방어적으로 전환

## 5. 데이터 수집 개선 방향

현재는 데이터를 매번 새로 수집하는 방식이 번거롭다. 앞으로는 수집기를 증분 수집 방식으로 개선한다.

추가할 기능:

```text
python collect_bybit_1m.py --days 365
```

- 지정한 기간만큼 과거 데이터를 수집한다.

```text
python collect_bybit_1m.py --update
```

- 기존 data/raw/BTCUSDT_1m.csv를 확인한다.
- 마지막 timestamp를 읽는다.
- 그 다음 시점부터 현재까지의 새 데이터만 수집한다.
- 기존 CSV에 append한다.
- 중복 timestamp 제거, 정렬, 저장을 수행한다.

```text
python collect_bybit_1m.py --watch
```

- 나중에 추가할 실시간 수집 모드다.
- 프로그램을 켜두면 1분마다 새 캔들을 수집한다.
- 실시간 대시보드와 연결할 수 있다.

우선순위는 --days와 --update가 먼저다. --watch는 실시간 대시보드를 만들 때 추가한다.

## 6. 뉴스 데이터 연동 방향

뉴스 데이터는 차트 데이터만으로 반영하기 어려운 외부 이벤트를 감지하기 위해 사용한다.

뉴스 AI가 판단할 항목:

- BTC 관련성 점수
- 시장 전체 관련성 점수
- 심각성 점수
- 방향성: positive / neutral / negative
- 변동성 유발 가능성
- 급락 위험 가능성
- 영향 시간대: short / medium / long
- 추천 리스크 액션: normal / caution / no_trade / reduce_position

뉴스 데이터 저장 예시:

```text
timestamp
source
title
summary
url
btc_relevance_score
severity_score
sentiment
market_impact
volatility_risk_score
risk_action
```

주의점:

- 뉴스 긍정이면 매수, 부정이면 매도 같은 단순 로직은 피한다.
- 뉴스는 방향성보다 변동성/불확실성/위험도 판단에 더 적합하다.
- 차트 리스크와 뉴스 리스크를 합쳐 최종 risk score를 계산한다.

## 7. 멀티호라이즌 예측 방향

사용자가 원하는 예측 단위는 다음과 같다.

- 15분봉 예측
- 1시간봉 예측
- 1일봉 예측

단, 이것을 가격 예측으로 정의하지 않는다.

더 적합한 정의는 다음과 같다.

### 15분 예측

- 초단기 방향성
- 단기 변동성
- 단기 진입 위험

### 1시간 예측

- 고변동 가능성
- 급락 위험
- 데이트레이딩 진입 필터

### 1일 예측

- 시장 국면
- 스윙 매매 환경
- 리스크 온/오프 판단

## 9. 백테스트 평가 기준

이 프로젝트의 평가는 단순 수익률만으로 하지 않는다.

중요한 평가 기준:

- total return
- buy and hold 대비 성과
- maximum drawdown
- volatility
- Sharpe-like ratio
- market exposure
- trade count
- fee impact
- high volatility 구간 회피 여부
- 급락 구간 회피 여부
- 포지션 축소가 MDD를 줄였는지

특히 고변동 모델은 수익률을 직접 올리는 모델이 아니라 리스크를 줄이는 모델로 평가해야 한다.
